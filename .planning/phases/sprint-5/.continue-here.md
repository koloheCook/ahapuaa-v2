## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Edit swallows declaration | When inserting `updateHUD()` after `checkTechUnlocks(state)`, the edit replaced the entire `const newTechs = checkTechUnlocks(state);` line, leaving `newTechs` undefined. The placement handler silently failed (ReferenceError) while the End Turn handler still worked. | advisory | When moving a call earlier/later in a block, use two separate edits: one to add the new line, one to remove the old. Never replace a declaration+call in a single string match. |
| Multiple sequential edits to same block | Made 3+ edits to the same pointerdown handler across different turns, creating ordering bugs that were hard to detect without re-reading the file. | advisory | After 2 edits to the same function, re-read the full function before making a third. |
