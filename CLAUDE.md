## Project

Ahupuaa: The Living Land is a browser-based strategy simulation of Hawaiian land
stewardship. Players manage a mountain-to-sea land division, balancing resources,
population, and cultural ike through traditional agriculture and aquaculture.
Water is the primary carrier of value. This is a Phaser 4 rebuild of a working
Vanilla JS prototype. Primary build going forward (ahupuaa-game Phaser 3 prototype
paused 2026-05-25 -- see that repo's README for context).

**Last shipped:** Sprint 4 complete - dual-clock system, Lono multiplier, ʻIke accumulation, HUD wired; 7/7 verified, browser-approved (2026-05-29)

## Coordinate System -- READ BEFORE WRITING ANY TILE POSITION CODE

**The canonical coordinate system is Tiled stagger-offset (col, row).**

- col 0-19, left to right
- row 0 = mauka (mountain top), row 55 = makai (ocean)
- Odd rows shift right by tileWidth/2 (Tiled stagger-index: odd)

### The only correct Tiled-to-pixel function

```js
// Returns { x, y } pixel center of tile (col, row)
const worldXY = map.tileToWorldXY(col, row);
```

Do not compute hex pixel centers manually. Do not route through rex board.
This function is confirmed correct by Spike B across all tested tile positions.

### Pixel center formula (for reference only -- use map.tileToWorldXY in code)

```
// tileWidth = 54, tileHeight = 46, hexsidelength = 23
even row: cx = col * 54 + 27,      cy = row * 34.5 + 23
odd row:  cx = col * 54 + 54,      cy = row * 34.5 + 23
```

### Internal tile data indexing

```js
tiles[col][row]  // COLUMN-FIRST. Locked from prototype. Do not invert.
```

### What "Even-O" means in this project

The prototype documentation calls this "Even-O Offset." This is a misnomer --
the actual behavior (odd rows shift right) matches Tiled's stagger-index: odd,
which is conventionally called Odd-O. Do not rename anything. The `getNeighbors()`
math is correct and empirically verified. The label is wrong; the code is right.

### Tiled object property format (Phaser 4.1.0)

Tiled object `properties` come as an array of `{name, type, value}` objects,
NOT a flat key-value map. Always use a getProp() helper:

```js
function getProp(obj, name) {
  if (!obj.properties) return undefined;
  if (Array.isArray(obj.properties)) {
    return obj.properties.find(p => p.name === name)?.value;
  }
  return obj.properties[name];
}
```

## The One Architectural Rule

`/src/game/` has zero Phaser imports. If a file in `/src/game/` imports
from Phaser, it is in the wrong directory. No exceptions.

Phaser scenes (in `/src/scenes/`) read from `/src/game/` but game logic
never reaches into Phaser. This is the boundary that makes the game logic
testable and the renderer replaceable.

## Advisory Mode

When proposing an architectural pattern, approach, or process decision, name
the industry term and explain why it applies.

Key terms active in this project:

- **Separation of concerns** -- `/src/game/` zero Phaser imports enforces
  this boundary between game logic and rendering engine
- **Spike** (TPM term) -- a time-boxed experiment to reduce uncertainty before
  committing to implementation; Spikes A, B, and C are examples. Spike A failed
  and changed Sprint 1 scope.
- **Canonical source of truth** -- `map.tileToWorldXY` is canonical for
  coordinate lookup; `getNeighbors()` is canonical for neighbor logic;
  `CONTENT.md` is canonical for Hawaiian terms. One owner per domain.
- **Immutable state pattern** -- `riverSystem.js` functions return new state
  objects rather than mutating; this makes god-event side effects traceable
- **Feature flag** -- `DEV_MOCK`, `devHideForest`, `debugMode` in prototype;
  keep this pattern in Phaser for demo control
- **Idempotency** -- `activatePath()` checks before adding; `blockSegment()`
  checks before adding; calling twice is safe
- **Blast radius** -- scope discipline limits how much a session failure damages
  the broader build; each sprint has a defined done condition so partial
  completion is still shippable
- **Check Phaser 4 first (canvas effects rule)** -- before writing custom JS
  for anything on the Phaser canvas (tints, tweens, timers, particles, audio,
  input), check Phaser 4's API for a native solution first. The game-clock
  timer (`this.time.delayedCall`) is preferred over `setTimeout`; Phaser tweens
  are preferred over manual tint-restore loops. Scope: canvas-side effects only.
  HTML-for-UI remains the architectural rule -- selector panels, tooltips, and
  status zones are HTML, not Phaser game objects.

## Hawaiian Term Integrity

All Hawaiian strings rendered in UI must be sourced from `CONTENT.md` (repo root).
Do not generate Hawaiian terms. Do not guess diacriticals.
Correct characters: ʻokina (U+02BB) and macrons (ā, ē, ī, ō, ū).
If a term is needed and not in `CONTENT.md`, stop and flag it -- do NOT invent it.
`CONTENT.md` is the cultural verification baseline for this project and grows over
time as new terms are researched and confirmed.
